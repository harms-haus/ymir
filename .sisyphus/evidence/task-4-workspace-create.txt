## 8. Critical Findings

1. **Serialization Format Incompatibility**: The Rust enum uses "content = "data"" which serializes structs as arrays, but the manual TypeScript protocol.ts expects named fields.

2. **Existing Tests Use Named Fields**: The protocol.test.ts and other TypeScript code use the named field format, not the array format.

3. **Fixture Validator Accepts Both**: The fixtureValidator only checks the message type, not the full structure. This is intentional for flexibility.

4. **ts-rs Generated Type**: The generated type correctly matches the Rust struct field names (camelCase), but doesn't include the message wrapper.

## 9. Recommendations

This format mismatch should be addressed in future tasks:
- Option 1: Change Rust to use externally tagged enum (remove content = "data")
- Option 2: Update TypeScript to handle array format from Rust
- Option 3: Keep both formats and add type adapters
