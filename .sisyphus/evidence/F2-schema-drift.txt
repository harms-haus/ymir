## Known Type System Differences

The following differences exist between the **generated types** (ts-rs output) and the **manual types** (apps/web/src/types/generated/protocol.ts). These are **architectural differences**, not drift:

### Semantic Drift (Design Choices)

| Aspect | Generated (ts-rs) | Manual (protocol.ts) | Impact |
|--------|-------------------|---------------------|--------|
| Type discriminators | Not included | Included (`type: 'X'`) | Expected - handled by Rust enum wrapper |
| Optional fields | `T \| null` | `T?` (optional) | Minor - both serialize same way |
| Timestamps | `bigint` | `number` | Minor - JavaScript handles both |

### Actual Schema Drift (Requires Resolution)

| Type | Field | Generated | Manual | Action Needed |
|------|-------|-----------|--------|---------------|
| AgentStatus | enum values | `"working" \| "waiting" \| "idle"` (PascalCase) | `'idle' \| 'working' \| 'waiting' \| 'error'` (lowercase + 'error') | **YES** - Case mismatch + missing 'error' |
| WorkspaceData | settings | `string \| null` | `Record<string, unknown>?` | **YES** - Type mismatch |
| WorktreeData | status | `string` | `'active' \| 'inactive' \| 'orphaned'` | **YES** - String vs union type |

## Sample Type Comparisons

### AgentStatus

**Generated (crates/ws-server/bindings/AgentStatus.ts)**:
```typescript
export type AgentStatus = "working" | "waiting" | "idle";
```

**Manual (apps/web/src/types/generated/protocol.ts)**:
```typescript
export type AgentStatus = 'idle' | 'working' | 'waiting' | 'error';
```

**Drift**: Case mismatch (PascalCase vs lowercase), missing 'error' value

### WorkspaceData

**Generated (crates/ws-server/bindings/WorkspaceData.ts)**:
```typescript
export type WorkspaceData = { 
  id: string, 
  name: string, 
  rootPath: string, 
  color: string | null, 
  icon: string | null, 
  worktreeBaseDir: string | null, 
  settings: string | null,  // ← string
  createdAt: bigint,        // ← bigint
  updatedAt: bigint,        // ← bigint
};
```

**Manual (apps/web/src/types/generated/protocol.ts)**:
```typescript
export interface Workspace {
  id: string;
  name: string;
  rootPath: string;
  color?: string;
  icon?: string;
  worktreeBaseDir?: string;
  settings?: Record<string, unknown>;  // ← object
  createdAt: number;                   // ← number
  updatedAt: number;                   // ← number
}
```

**Drift**: settings type, timestamp type, optional handling

### WorktreeData

**Generated (crates/ws-server/bindings/WorktreeData.ts)**:
```typescript
export type WorktreeData = { 
  id: string, 
  workspaceId: string, 
  branchName: string, 
  path: string, 
  status: string,  // ← untyped string
  createdAt: bigint
};
```

**Manual (apps/web/src/types/generated/protocol.ts)**:
```typescript
export interface Worktree {
  id: string;
  workspaceId: string;
  branchName: string;
  path: string;
  status: 'active' | 'inactive' | 'orphaned';  // ← typed union
  createdAt: number;
}
```

**Drift**: status field lacks type safety in generated version

## Recommendations

### 1. No Immediate Action Required for Generated Bindings

The ts-rs generated bindings are in sync with the Rust codebase. No regeneration needed.

### 2. Type System Alignment (Future Work)

To fully align manual and generated types:

1. **AgentStatus**: Add `#[serde(rename_all = "lowercase")]` to Rust enum, add `error` variant
2. **WorkspaceData.settings**: Change Rust type from `Option<String>` to `Option<serde_json::Value>`
3. **WorktreeData.status**: Create a typed enum in Rust with `#[derive(TS)]`
4. **Timestamps**: Use `i64` in Rust instead of `usize` for better JavaScript interop
